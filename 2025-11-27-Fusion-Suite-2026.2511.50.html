Mer info om JAWS 2026.2511.146:

JAWS 2026.2511.146

NEW

  • Added support on ARM64 devices for the DotPad.
  • Based on your feedback, the setting for Enable SayAll in Outlook/Windows Mail when opening an email is once again on by default.
  • Updated the installed .NET Redist, VCRedist, and WebView2 to latest versions.

Fixed

  • JAWS does not activate forms mode when the select edit field dialogue box is used to move to an editable combo box. 138441 (Enes Saribas)
  • No cursor indicator in Braille when JAWS Cursor invoked from within Virtual Viewer. 139239 (Howard Goldstein)
  • JAWS is not announcing the selected state of the row in table (Microsoft Azure Stack Development kit). 123787 (Michelle Ruth Christensen)
  • Touch Cursor accidentally provides means to access hidden Account Tab in About dialog.
  • JAWS does not output Braille for WPF TextBlock controls.
  • Install JAWS on clean image and the Sign-in prompts that you have just a few minutes left to sign in.
  • Cannot configure Alva BC 6 drivers with JAWS.
  • If a string is assigned literal text and the initial quote is missing, the compiler crashes.
  • While editing a document in Google Docs with Monarch, it is no longer possible to pan through text you've typed.
  • Google Docs: Nested list levels for checkable lists are indicated in speech as you enter them but not as you move up/down by lines through the document.
  • Chrome: JAWS presents the radio button on a separate line in the virtual buffer from its associated label based on CSS.
  • Chrome: When iFrame Is Used To Insert Another HTML Document, JAWS speaks the Full Path and File Name.
  • Chrome:Aria-Orientation tag handled inconsistently.
